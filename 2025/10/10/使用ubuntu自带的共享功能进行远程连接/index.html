使用ubuntu自带的共享功能进行远程连接

发表于 2025-10-10 更新于 2026-06-04 分类于 Linux运维 阅读次数：
本文字数： 2.9k 阅读时长 ≈ 3 分钟

通过路由器的端口映射功能，实验室可以将服务器的任意端口映射到路由器的指定端口，从而实现外部网络对内部局域网服务器的访问。同时，结合Ubuntu自带的共享功能和Windows的远程桌面连接功能，可以实现远程控制，提供低延迟和高分辨率的屏幕显示体验。

我曾使用过多种远程控制软件，包括 ToDesk、向日葵、TeamViewer 和 AnyDesk。但在实际使用中，我遇到了一些问题。ToDesk 和向日葵的延迟较高，使用体验不理想；而 AnyDesk 开始收费，这让我无法继续使用。剩下的选择是 TeamViewer，但其登录过程繁琐，且延迟也相对较高。

在实验室师兄的推荐下，我决定尝试配置 Ubuntu 的 RDP 远程连接。通过 Ubuntu 自带的共享功能，我能够有效地进行远程桌面操作，而且不再需要依赖收费或不稳定的第三方软件。

原理

Ubuntu（Ubuntu22.04以后版本）自带RDP远程桌面，微软的 远程桌面协议（RDP） 是一个允许从一台计算机到另一台计算机进行图形化远程桌面连接的协议。RDP 的工作原理是让一台主机运行软件，允许其他几台计算机连接到它。

配置过程

ubuntu被控端设置：

  1. 打开系统设置
  2. 然后从左侧导航到“共享”，然后打开应用标题右上角的切换图标。最后点击“远程桌面”（Remote Desktop）
  3. 配置用户、密码

Windows端设置：
使用 RDP 协议，Windows 用户可以简单地从开始菜单搜索并打开内置的“远程桌面连接”应用程序。然后输入 Ubuntu 22.04 的计算机名称或 IP 地址进行连接。

禁用wayland协议

Wayland 是一种显示服务器协议，它主要用于在操作系统中处理图形显示相关的事务。启用该协议会导致一些问题，例如图形界面渲染错误、远控软件、某些硬件不兼容等，网上也有说因这个协议XRDP黑屏、向日葵、腾讯会议连接失败，例如1，2，3

禁用方法很简单，找到/etc/gdm3/custom.conf，将WaylandEnable=false的注释井号键#取消掉

走过的弯路

ubuntu 20.0.4

ubuntu20.0.4未自带RDP，需要安装XRDP ，其是 RDP 的一个开源实现，不需要运行任何专有程序。XRDP 不仅试图遵循 RDP，而且还与常规的 RDP 客户端兼容，如 Remmina 和 GNOME Boxes。但在安装过程中也有许多麻烦(建议直接备份后升级ubuntu系统)

XRDP安装

sudo apt install xrdp

黑屏问题

本机没有注销的话，远程桌面就会黑屏—–>退出本地登录，也就是注销登录
编辑 /etc/xrdp/startwm.sh 文件：

  1. 打开文件
    sudo vim /etc/xrdp/startwm.sh
  2. 添加配置
    unset DBUS_SESSION_BUS_ADDRESS
    unset XDG_RUNTIME_DIR
  3. 重启xrdp服务
    sudo systemctl restart xrdp.service
