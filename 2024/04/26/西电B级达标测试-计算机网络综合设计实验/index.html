实验拓扑搭建

首先，我们根据实验给出的拓扑图及子网掩码、网络号等参数在eSNP软件上选择相应的设备构建网络拓扑图，交换机我们选择地设备上S5700，路由器我们选择AR2220。连接好设备后，开启设备进行试验。

任务一，为A和B主机配置DHCP

首先双击打开PC1和PC2，在基础设置界面地IP地址设置中选择dhcp，之后我们进行与主机A和主机B相连的路由器AR1的配置。进入AR1后，首先进入系统视图，使用命令dhcp enable开启dhcp服务。其次，我们在AR1上进行IP池的配置，设置掩码和网关。因为我们对网使用dhcp服务，所以我们的掩码选择24，网关为AR1的GE0/0/0接口IP为192.168.1.254，在GE0/0/0接口上使用命令dhcp select global，开启dhcp全局分配。

在完成了dhcp服务的配置之后，为了实现四个PC各自能够ping通，我们再对路由器的GE0/0/1接口和路由表进行配置。我们将GE0/0/1的IP地址设置为192.168.5.2，目的是网络号192.168.1.0的设备经过AR1能直接到192.168.5.0网段。最后我们配置静态路由表，对于AR1其目的网络是最右侧的网3，该条静态路由的目的是沟通网1和网3，而下一跳的IP地址则是AR2中GE0/0/0接口的IP地址，于是我们设置到达目的地址192.168.3.0的下一跳IP地址为192.168.5.254。

对主机A进行限速

根据题目要求，我们要对PC1所连接的网络设备端口进出入口进行限速，限制在端口最大速率的50%。将其对应到实验的网络拓扑图，即LSW1中的GE0/0/0接口。首先，我们需要进入LSW1的命令行界面，通过指令进入系统视图并进入GE0/0/0接口视图。我们使用如下命令完成对接口的限速：qos lr inbound/outbound cir xxx cbs xxx。

在对指令的解释之前，我们首先需要了解GE和E的区别。GE是千兆(1000MB)以太网口，E是百兆(100MB)以太网口，在此我们使用千兆以太网口。在指令中，qos是服务质量，lr是信息速率，inbound是入口方向，outbound是出口方向，cir是承诺信息速率即限速后的速率，cbs是承诺突发大小即突发情况下能通过的最大速率。

本实验中我们使用千兆以太网接口，但从接口角度其速率为1000MB=1000*1024Kb，对其限速到50%即0.5*1000*1024Kb=512000kb，但是PC段速率为百兆，取较慢段速率的50%为5*100*1024Kb=51200kb。到此，我们已经完成了任务三，完成了对PC1的限速。

任务四：ARP攻击的防护

通过指令arp speed-limit source-ip xxx maximum xxx进行ARP限速。arp speed-limit为配置ARP速率限制的操作，source-ip xxx设置限制的源IP地址，即限制此IP的ARP请求，maximum xxx设置最大的请求或响应数。这个指令的作用是限制指定源IP地址发出的ARP请求或响应的数量，以防止ARP泛洪攻击或其他恶意行为对网络造成过大的负载或干扰。

通过指令arp anti-attack rate-limit 50 1对端口进行限速并完成防护，arp anti-attack rate-limit配置了ARP反攻击速率限制的操作，50表示限制为50个ARP请求或响应，1表示时间间隔。这个指令的作用是限制在1秒内收到ARP请求或响应的数量不超过50个 。

通过指令arp anti-attack entry-check fixed-mac enable进行ARP表项固化，arp anti-attack指示了配置ARP反攻击功能的操作，entry-check表示对ARP表中的条目进行检查，以确保它们的合法性，fixed-mac表示对于已知的合法ARP条目，将固定MAC地址，enable表示启用上述设置。这个指令的作用是在ARP反攻击功能中启用条目检查，并对已知的合法ARP条目固定MAC地址，以提高网络安全性，防止ARP欺骗攻击等恶意行为

我们通过指令arp anti-attack gateway-duplicate enable进行ARP防网关冲突，其中gateway-duplicate表示检测网关地址的重复性。这个指令的作用是在ARP反攻击功能中启用对网关地址的重复检测。ARP反攻击功能会监测网络中的ARP消息，以检测是否有多个设备宣称拥有相同的网关IP地址，从而帮助防止ARP欺骗攻击。

静态路由和动态路由的优缺点

·静态路由的优点

  1. 占用的CPU和内存资源少
  2. 可控性强，便于管理员了解整个网络路由信息。
  3. 不需要动态更新路由，可以减少对带宽的占用，提高网络安全性。
  4. 配置简单

·静态路由的缺点

  1. 初始配置和维护耗费管理员大量时间。
  2. 配置时容易出错，尤其对于大型网络
  3. 当网络拓扑发生变化时，需要管理员手动维护
  4. 随着网络规模的增长和配置的扩展，维护越来越麻烦。
  5. 需要管理员对整个网络的情况完全了解后才能进行恰当的操作。

·动态路由的优点

  1. 当增加或删除网络时，管理员维护路由配置的工作量少
  2. 当网络拓扑结构发生变化时，路由协议可以自动做出调整来更新路由表
  3. 配置不容易错。
  4. 扩展性好，网络规模越大，越能体现出动态路由的优势。

·动态路由的缺点

  1. 由于需要相互交换路由信息，需要占用额外的资源，如cpu时间和内存以及链路带宽等。
  2. 需要掌握更多的网络知识才能进行配置、验证和故障排除等工作，特别是一些复杂的动态路由协议对管理员的要求相对较高。
  3. 安全性不如静态路由。

动态路由有哪些

Ospf和rip域内，bgp域间（计网第22章）

OSPF（Open Shortest Path First）：OSPF 是一种开放式的链路状态路由协议，通过交换链路状态信息来计算最短路径，并基于此更新路由表。OSPF 适用于大型企业网络和互联网服务提供商网络。

RIP（Routing Information Protocol）：RIP 是一种基于距离向量的路由协议，使用跳数作为路径选择的度量标准。虽然 RIP 简单易用，但由于其限制，如最大跳数限制和较慢的收敛速度，通常在小型网络中使用。

BGP（Border Gateway Protocol）：BGP 是一种路径矢量路由协议，主要用于互联网核心路由器之间的路由信息交换。BGP 不仅可以在自治系统之间交换路由信息，还可以应用于大型企业网络的边界路由器之间的路由控制。

dhcp的原理

  1. 客户机以广播的形式向服务端发送DHCP discover信息

  2. 服务端收到消息后确认有可用的合法ip地址就会把IP地址加入到DHCP offer中广播出去。

  3. 客户机收到广播后选择并请求租用ip地址

  4. 服务端回应确定租约

限速与ARP

问题一：对PC1是在哪配置限速的。

答： 在LSW1中对应的PC1的端口设置。

问题二：解释ARP泛洪攻击的原理。

答：ARP泛洪攻击主要有两种。第一种是攻击者发出大量能解析但无效的ARP请求，导致网关无法正常响应，从而造成网关部分的CPU利用率上升难以响应正常服务请求。一种是攻击者发送大量不能被解析的ARP请求，网关会被错误的ARP表充满导致无法更新维护正常ARP表，消耗网络带宽资源。
